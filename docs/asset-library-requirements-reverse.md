# 素材库模块需求文档（反向整理）

## 0. 文档说明
- 文档类型：反向需求文档（根据现有实现整理）。
- 目标模块：素材库（`assets` 标签页）。
- 依据代码：
- `src/components/AssetLibrary.tsx`
- `src/data.ts`
- `src/types.ts`
- `src/App.tsx`
- 说明：本文描述的是“当前实现行为”，不是未来理想态规划。

## 1. 模块目标
- 为美术生产提供统一素材浏览、目录管理、筛选检索、离线下载、DCC 导入能力。
- 在个人空间提供上传入库能力，并在上传过程中提供自动打标与人工修订。
- 通过日志和状态反馈降低用户在下载、导入、上传过程中的不确定性。

## 2. 适用范围
- 支持空间：
- 项目空间 A：可查看项目素材、下载、导入 DCC。
- 个人空间：可查看个人素材、目录管理、上传入库。
- 暂不支持空间：
- 项目空间 B、全员共享空间：显示“无可用素材”空状态。

## 3. 信息架构与数据规模
- 素材分类：
- 全部、角色原画、场景原画、角色模型、场景模型、动画序列、粒子视频、GUI 切图。
- Demo 数据规模（项目空间 A）：
- 素材总数 60。
- 分类分布：动画 10、角色原画 8、角色模型 11、GUI 8、场景原画 6、场景模型 10、视频 7。
- 默认目录结构：
- 根目录：`一级文件夹`（`folder-primary`）。
- 二级目录：`二级文件夹`、`二级文件夹01`、`003`、`004`、`005`、`006`、`007`。
- 三级示例：`武将参考`、`武器道具`、`赤壁战场`、`洛阳建筑`。

## 4. 核心流程
- 流程 A（浏览检索）：选目录 -> 选分类 -> 关键词检索 -> 分页浏览 -> 打开详情。
- 流程 B（离线下载）：详情页点击下载（仅未下载素材） -> 下载前本地存储空间（磁盘）校验 -> 进入并发队列 -> 下载完成入本地缓存（已下载素材直接定位本地目录）。
- 流程 C（DCC 导入）：详情页选择 DCC -> 连接与格式校验 -> 建立流式导入通道 -> 执行导入/模式配置 -> 导入成功提示。
- 流程 D（个人上传）：上传入口 -> 文件校验 -> AI 自动打标 -> 用户改标签 -> 确认入库到置顶目录。

## 5. 功能需求清单（反向）
| 编号 | 功能 | 需求描述 |
| --- | --- | --- |
| FR-01 | 空间隔离 | 素材库按当前空间展示数据，仅项目空间 A 与个人空间有内容。 |
| FR-02 | 目录树浏览 | 支持多层级目录树展示、展开/收起、选择目录、显示每个目录素材计数。 |
| FR-03 | 目录检索 | 支持按目录名称搜索，命中目录时需保留其祖先链路以便定位。 |

| FR-05 | 删除回收策略 | 删除目录时，目录下素材自动迁移到回收目标目录（优先父目录，否则根目录）。 |
| FR-06 | 子目录素材开关 | 支持“显示子文件夹素材”开关，控制当前目录是否包含后代目录素材。 |
| FR-07 | 分类筛选 | 支持分类 Tab 筛选。 |

| FR-09 | 分页 | 每页显示 12 条素材，支持上一页/下一页与页码跳转。 |
| FR-10 | 素材详情 | 项目空间素材支持打开详情弹层，展示预览、描述、格式、大小、分类、作者、标签。 |
| FR-11 | 本地下载 | 支持将素材下载到本地缓存；已下载素材可直接定位本地目录。 |
| FR-12 | 下载并发队列 | 同时最多 3 个下载任务；超出任务进入排队并自动晋升。 |
| FR-13 | 下载取消 | 在详情态可将排队任务移出缓冲队列。 |
| FR-14 | DCC 导入前置校验 | 导入前必须校验 DCC 连接状态与素材格式兼容性。 |
| FR-15 | 流式导入直达 | 一键导入走流式传输链路，不依赖离线下载与本地缓存前置。 |
| FR-16 | 可选导入模式 | Photoshop/Blender/Maya/3ds Max 在特定格式下需用户选择导入模式后执行。 |
| FR-17 | 个人空间上传入口 | 仅个人空间显示“上传素材”入口。 |
| FR-18 | 上传格式限制 | 仅支持图片、GIF、视频。 |
| FR-19 | 上传批量限制 | 单次最多 500 文件，且总大小不超过 10GB。 |
| FR-20 | AI 自动打标 | 上传后自动提取标签（类型、格式、尺寸/时长、文件名语义、体积标签）。 |
| FR-21 | 标签人工修订 | 支持新增、编辑、删除标签后再确认入库。 |
| FR-22 | 上传入库规则 | 确认上传后素材进入个人空间，并写入置顶目录（`folder-primary`）。 |
| FR-23 | 本地持久化 | 目录结构、素材目录归属、目录面板宽度需本地持久化。 |

## 6. 关键业务规则
### 6.1 目录规则
- 至少保留一个目录，不允许删除到“零目录”。
- 新建目录 ID 采用时间戳格式（`folder-<timestamp>`）。
- 目录排序中，内置目录优先，用户新建目录按创建时间排序。
- 为兼容旧数据，若命中已废弃目录 ID（`folder-browser`、`folder-cloud-local`），自动归并到 `folder-primary`。

### 6.2 下载规则
- 若素材已下载：不再执行本地存储空间（磁盘）校验，直接定位本地目录。
- 下载前校验可用磁盘空间：`asset.sizeMB / 1024 <= simulatedDiskGB`。
- 下载进度为模拟进度（每 200ms 增长 10% 直到 100%）。
- 下载成功后：
- 写入已下载集合 `downloadedAssetIds`。
- 可用磁盘按素材大小扣减（保底 0.1GB）。

### 6.3 DCC 导入规则
- 导入前置：目标 DCC 状态必须为 `Connected`。
- 导入链路：采用流式导入，素材数据直接传输至目标 DCC。
- 流程约束：流式导入不触发离线下载队列，不要求素材处于“已下载”状态，不执行本地存储空间（磁盘）校验。
- 兼容矩阵：
- ComfyUI：`png/jpg/jpeg/mp4/mov`
- Photoshop：`png/jpg/jpeg`
- Blender：`fbx/obj/blend/png/jpg/jpeg`
- Maya：`fbx/obj/ma/mb/png/jpg/jpeg`
- 3ds Max：`fbx/obj/max/png/jpg/jpeg`
- 可选模式触发规则：
- Photoshop：全部格式均弹模式选择（图层置入/独立画板）。
- Blender/Maya/3ds Max：非图片格式弹模式选择；图片格式直接导入。

### 6.4 上传规则
- 上传前必须通过格式校验、数量校验、体积校验。
- 自动打标完成前不可“确认上传”。
- 自动打标每条初始最多保留 8 个标签。
- 人工修订后每条最多保留 12 个标签。
- 入库后分类映射：
- 视频文件 -> `Video`
- 图片/GIF -> `GUI`

## 7. 状态与存储
- 关键运行态：
- 当前目录、分类、关键词、页码、详情弹层、下载任务队列、上传草稿、标签编辑输入、目录面板宽度。
- 本地存储键：
- `art-launcher-asset-folders-v1`
- `art-launcher-asset-folder-assignments-v1`
- `art-launcher-folder-pane-width-v1`

## 8. 交互与反馈要求
- 用户触发关键动作应有反馈：日志输出、弹窗提示、进度条、状态标识（已下载/排队/下载中/已打标）。
- 素材详情弹层支持 `Esc` 快捷关闭。
- 目录右键菜单在点击空白处/滚动时自动关闭。

## 9. 非功能要求（按当前实现归纳）
- 响应式：桌面端支持目录面板拖拽宽度；非桌面端自动关闭拖拽态。
- 性能：列表分页展示，降低一次性渲染压力。
- 可用性：下载、导入、上传都具备阻断条件下的明确提示。
- 可追踪性：关键行为通过统一 `addLog` 记录。

## 10. 已知限制与后续可演进项
- 当前下载与导入为模拟流程，尚未接入真实后端任务系统。
- 个人空间素材未提供与项目素材一致的“详情-下载-导入”操作链路。
- 上传生成的 `ObjectURL` 缺少统一回收策略，长时间使用可能有内存占用风险。
- 目录与归属关系存储在本地浏览器，缺少多端同步能力。
