## 1. 模块定位与上下文

### 1.1 平台定位
平台是"游戏美术团队的 DCC 工具 / 插件扩展 / 美术素材管理平台"，提供：安装、连接、启动监控，以及**共享项目空间素材**功能。
素材库模块是其中面向"美术资产消费与入库"的核心功能区。

### 1.2 模块在产品中的位置
- 导航入口：左侧边栏（Sidebar）的「素材」页签（`currentTab === 'assets'`）。
- 顶层上下文：受左侧边栏"项目空间切换"控制；当前处于哪个空间，决定素材库呈现的内容与可用能力。
- 关联模块：
  - **Sidebar**：负责空间切换（SpaceId）。
  - **AppManager**：负责 DCC 的连接状态（`AppStatus.Connected`），导入功能强依赖该状态。
  - **SettingsPanel**：负责磁盘可用容量（`simulatedDiskGB`）与"一键清理临时缓存"，
    直接影响下载是否被阻断，以及清理后能否恢复下载。

## 3. 多空间模型（Space）

### 3.1 空间定义
平台定义 4 个空间（`SpaceId`）：

| SpaceId | 名称 | 素材库是否有内容（V1） |
|---|---|---|
| `projectA` | 三国：冰河时代 | ✅ 预置 60 条素材 + 默认文件夹结构 |
| `projectB` | 项目空间 B（赛博朋克动作） | ❌ V1 暂无内容 |
| `shared` | 全员共享空间 | ❌ V1 暂无内容 |
| `personal` | 个人空间（如"慕也的个人空间"） | ⚪ 初始为空，由用户上传积累 |

### 3.2 各空间素材库行为差异

| 能力 | 项目空间 A | 项目空间 B / 共享空间 | 个人空间 |
|---|---|---|---|
| 素材列表 | 预置资产 | 无 | 用户上传资产 |
| 文件夹目录树 | 默认结构 + 可自定义 | 不展示目录树（空态页） | 与项目空间一致的目录逻辑 |
| 素材浏览/检索/分类/分页 | ✅ | ❌（空态） | ✅ |
| 素材详情弹窗 | ✅ | ❌ | ❌（个人空间卡片不可点开详情） |
| 本地下载 / DCC 导入 | ✅ | ❌ | ❌ |
| 上传素材 | ❌ | ❌ | ✅（仅个人空间显示「上传素材」按钮） |
| 切换空间副作用 | — | 提示"无预置缓存，清单已清空" | — |

### 3.3 空间切换规则
- 切换空间时：重置当前页码到第 1 页、关闭已打开的素材详情弹窗。
- 切到非项目空间 A 时，记录日志提示"该空间无预置缓存/清单已清空"。
- 项目空间 B / 共享空间：素材库主体区域展示**空态页**，提示该空间暂无资产，
  并引导用户切回"项目空间 A"。

## 5. 功能需求

### 5.1 文件夹目录树管理

| 需求 | 说明 |
|---|---|
| 树形层级 | 支持任意层级父子文件夹；展开/收起子节点（默认展开根、角色、场景三个目录） |
| 新建文件夹 | 可新建"一级文件夹"或"子文件夹"（右键菜单"新建同级/子文件夹"、面板顶部"+"新建一级） |
| 重命名 | 右键菜单 → 重命名（内联输入框，回车确认 / Esc 取消） |
| 删除 | 右键菜单 → 删除文件夹；**连带删除其所有子孙文件夹**；**允许删至 0 个**（文件夹目录可为空） |
| 删除时素材归属 | 被删文件夹（含子孙）下的素材**移动到上级文件夹**；若删除后无任何文件夹留存，则**解除其目录归属**（素材暂不在目录中显示，直到新建文件夹） |
| 空目录态 | 文件夹数为 0 时：目录树显示"暂无文件夹，点击 + 新建"引导；素材区提示"请新建文件夹后查看"；素材数据不丢失，仅解除归属 |
| 删除二次确认 | 弹窗确认，提示将被删的子文件夹数量与受影响素材数量 |
| 右键上下文菜单 | 内容：新建同级文件夹 / 新建子文件夹 / 重命名 / 删除文件夹；菜单外点击或页面滚动自动关闭 |
| 目录搜索 | 搜索框过滤目录名；命中节点会连同其所有祖先路径一并保留可见（便于定位） |
| 文件夹素材计数 | 每个文件夹显示含子孙在内的素材总数 |
| 排序 | 默认/系统目录在前；用户新建目录按创建时间戳升序；其余按名称（中文拼音 + 数字感知）排序 |
| 面包屑路径 | 选中文件夹的完整祖先路径，可逐级点击跳转 |
| 持久化 | 文件夹结构、素材-文件夹归属关系、面板宽度均持久化到 `localStorage` |
| 数据自愈 | 读取本地存储时，自动剔除/迁移已废弃的默认目录 ID（`folder-browser`、`folder-cloud-local`），把其归属迁移到主目录 `folder-primary` |

#### 5.1.1 文件夹命名校验规则
新建 / 重命名提交时统一校验（实时反馈：输入即校验，不通过则禁用「确认」按钮 + 红色错误提示 + 拦截并写错误日志）：

| 规则 | 限制 |
|---|---|
| 非空 | 去首尾空格后不能为空 |
| 长度 | ≤ 32 字符（按字符数计数，超出禁止提交；输入框 `maxLength=32` 软拦截 + JS 硬校验） |
| 非法字符 | 禁止 `\ / : * ? " < > \|` 及控制字符（前瞻兼容真实文件系统落盘） |
| 前缀 | 不能以 `.` 开头（隐藏文件惯例） |
| 保留名 | 禁用 Windows 保留名：`CON / PRN / AUX / NUL / COM1–9 / LPT1–9`（大小写不敏感） |
| 同级重名 | 同一父目录下不允许同名文件夹（大小写不敏感比较，但保留用户输入的原大小写） |

错误文案：`文件夹名称不能为空` / `文件夹名称不能超过 32 个字符（当前 N 个）` / `名称不能包含 \ / : * ? " < > | 等特殊字符` / `名称不能以 "." 开头` / `不能使用系统保留名（如 CON、NUL、COM1 等）` / `该目录下已存在同名文件夹`。

### 5.2 素材浏览与检索

#### 5.2.1 分类筛选
固定 8 个分类 Tab（来自代码 `categoryTabs`，与 PRD 对齐）：

| Tab | 枚举值 |
|---|---|
| 全部 | `all` |
| 角色原画 | `char_concept` |
| 场景原画 | `scene_concept` |
| 角色模型 | `char_model` |
| 场景模型 | `scene_model` |
| 动画序列 | `animation` |
| 粒子视频 | `video` |
| GUI 切图 | `gui` |

#### 5.2.2 关键字搜索
- 搜索范围：素材**名称** + **标签**（大小写不敏感，包含匹配）。
- 与分类筛选、文件夹范围叠加生效（AND 关系）。

#### 5.2.3 文件夹范围与"显示子文件夹素材"
- 素材列表范围 = 当前选中文件夹（+ 可选子孙文件夹）。
- 「显示子文件夹素材」开关：开启时列出当前文件夹及其所有子孙文件夹下的素材；关闭时仅当前文件夹直挂素材。
- 切换分类 / 关键字 / 空间 / 文件夹 / 子文件夹开关时，页码重置为第 1 页。

#### 5.2.4 子文件夹区
- 在素材列表上方以卡片网格展示当前文件夹的**直属子文件夹**，点击进入。
- 卡片显示：文件夹图标、名称、素材总数、子目录数。
- 无子文件夹时展示空提示。

#### 5.2.5 分页
- 每页固定 12 条。
- 分页器：上一页 / 页码 / 下一页，并显示"当前 X–Y / 共 Z 项"、"页码 n / 总页数"。
- 仅在总页数 > 1 时显示分页器。

#### 5.2.6 素材卡片
- 缩略图（16:9）、格式角标（`.PNG` 等）、已下载勾标。
- 下载中/排队中状态遮罩（见 5.4）。
- 卡片信息（默认显示）：名称、类别、大小；项目空间下有"查看素材详情"入口。
- 个人空间：卡片**不可点开详情**（无详情弹窗）。

交互：

### 5.4 本地下载（F9 Path A：本地磁盘离线提取）

#### 5.4.1 下载触发与状态
- 入口：素材详情弹窗「下载到本地缓存 (XX MB)」按钮。
- 下载状态机：`queued`（排队）→ `downloading`（下载中，0–100%）→ 完成（加入"已下载"集合）。
- 卡片与详情弹窗均需展示下载进度与排队状态。

#### 5.4.2 并发队列规则（Max 3）
- 全局最多 **3 个**素材同时处于 `downloading`。
- 超出部分进入 `queued` 排队；当有下载槽位空闲时，按队列顺序自动提升为 `downloading`。
- 排队中的任务可被"移出缓冲区"取消。

#### 5.4.3 下载前置校验
| 校验 | 失败行为 |
|---|---|
| 已下载 | 不再下载，改为"在资源管理器中打开文件夹"（定位到本地目录） |
| 磁盘空间不足（所需 GB > 当前可用） | 阻断下载，提示去"设置"页清理临时文件或扩展配置 |
| 该素材已在下载/排队中 | 忽略重复触发 |

#### 5.4.4 下载完成
- 记入"已下载"集合；按素材体积扣减可用磁盘容量。
- 记录成功日志（含"本地安全隔离包定位完成"等措辞）。

> 【实现备注】下载为前端模拟：进度每 200ms +10%；磁盘容量为 `simulatedDiskGB` 模拟值，
> 默认 12GB（刻意设置成 < ComfyUI 安装所需 15GB，用于演示空间不足）。
> 真实落地需替换为真实下载 + 真实磁盘配额系统。

#### 5.4.5 本地目录约定
- 下载目录形如：`C:\Program Files\ArtPlatform\downloads\{assetId}\{name}.{format}`。
- "打开文件夹"即在 Windows 资源管理器中高亮定位该目录（当前为 `alert` 模拟）。

### 5.5 DCC 一键热导入（F9 Path B：专属通道智能热导入）

#### 5.5.1 触发与前置校验
1. DCC 必须处于「已连接」（`AppStatus.Connected`）；未连接则提示去"应用管理"启动并等待端口通讯变为"已连接"。
2. 素材须已下载到本地：
   - 未下载 → 自动入队下载，下载完成后**自动触发导入**（带 `targetDccImportAfterDownload` 钩子）。
   - 自动下载同样受磁盘空间校验约束。
3. 通过校验后执行导入。

#### 5.5.2 格式兼容性矩阵
导入按钮的可用性由"素材格式 × DCC"兼容矩阵决定。不兼容的 DCC 显示为禁用项并给出原因。

| DCC | 兼容格式 | 不兼容提示（示例） |
|---|---|---|
| ComfyUI | PNG / JPG / JPEG / MP4 / MOV | 仅支持 JPG/PNG 图像及 MP4/MOV 视频输入加载 |
| Photoshop | PNG / JPG / JPEG | 仅支持 JPG 和 PNG 经典平面栅格格式置入 |
| Blender | FBX / OBJ / BLEND + PNG/JPG/JPEG（作参考面） | 无法理解 .ma、.mb 或 .max 专有 DCC 场景网格 |
| Maya | FBX / OBJ / MA / MB + PNG/JPG/JPEG（作图像平面） | 不接收 .blend 及 .max，请预先转换 |
| 3ds Max | FBX / OBJ / MAX + PNG/JPG/JPEG（作视口背景） | 不支持 .ma/.mb、.blend，导入中止 |

#### 5.5.3 导入行为：直通 vs「可选」模式
- **直通导入**（无弹窗）：ComfyUI 全部、以及各 DCC 接收**平面图片**（PNG/JPG/JPEG）时，直接执行。
- **「可选」模式弹窗**（需用户选择实例化方式）：
  - Photoshop（图片）：方式一 智能对象图层置入 / 方式二 独立画板栅格化排放
  - Blender（模型）：方式一 直接本体追加导入（Append）/ 方式二 外部资源轻量引用（Link）
  - Maya（模型）：方式一 导入并入当前层（命名空间）/ 方式二 强制新建场景加载
  - 3ds Max（模型）：方式一 场景追加合并（Merge）/ 方式二 作为新项目文件打开
- 每种 DCC 的「可选」弹窗会预置一个默认选项。

#### 5.5.4 导入结果反馈
- 成功通知（`alert` + 日志）：说明素材、目标 DCC、投送模式，并提示可在 DCC 大纲/面板中调整。
- 投送模式文案映射：layer/artboard/append/link/importScene/newScene/merge/default → 各自中文模式名。

> 【实现备注】导入为前端模拟，仅产出日志与提示；真实落地需对接各 DCC 的端口/脚本通道。

### 5.6 个人空间素材上传与 AI 自动打标

#### 5.6.1 入口与说明弹窗
- 仅个人空间显示「上传素材」按钮。
- 点击先弹出"上传素材说明"，要点：
  1. 目录逻辑与项目空间一致（一级目录、子目录、目录搜索、右键管理）。
  2. 入库目录：确认上传后自动写入**个人空间置顶目录（一级目录）**。
  3. 格式限制：支持图片、动图(GIF)、视频。
  4. 批量限制：单次最多 500 条，总大小不超过 10 GB。
  5. 智能处理：上传阶段自动 AI 打标，支持标签增删改后再确认。

#### 5.6.2 文件选择与校验
- 支持多选；接受的类型：`image/*`、`video/*`、`.gif`。
- **格式校验**：仅支持图片 / GIF / 视频；含不支持文件 → 整批拦截并报错。
- **数量上限**：单次 ≤ 500 条；超出 → 整批拦截并提示分批上传。
- **体积上限**：单次总大小 ≤ 10 GB；超出 → 整批拦截。
- 通过校验后关闭说明弹窗，进入"上传预处理与 AI 自动打标"面板。

#### 5.6.3 上传预处理与 AI 打标面板
- 顶部进度：`已完成标注 X / 总数` + 总体进度条；处理中显示 spinner。
- 每个待上传项一行：预览（视频用 `<video>` 元数据预览）、文件名/源文件名/格式/大小/类型/打标状态。
- 每项的标签：可编辑（点标签改）、可删除（x）、可新增（输入回车）；标签去重、去 `#`、上限 12 个。
- AI 自动打标内容（见 5.6.4）逐条异步生成，状态从"打标中"→"已打标"。

#### 5.6.4 自动打标规则（本地启发式，命名为"AI 自动打标"）
每个文件自动生成标签，规则（去重后最多 8 个）：
- 类型标签：图片 / 动图 / 视频。
- 格式标签：扩展名大写（如 PNG）。
- 固定标签：`AI自动打标`。
- 文件名分词：中文连续 ≥2 字片段 + 英文/数字 token（长度 ≥2 且非纯数字），最多取 4 个。
- 体积标签：≥200MB → `大体积`；≤5MB → `轻量`。
- 尺寸/方向标签（按类型）：
  - 视频：读取宽高与时长 → `宽x高`；≤10s 短视频 / ≥60s 长视频 / 其余中视频。
  - 图片/GIF：读取宽高 → `宽x高`；宽>高 横版 / 宽<高 竖版 / 相等 方图。

> 【实现备注】当前"AI 打标"为**纯前端本地启发式**（文件名分词 + 浏览器读取媒体元数据），
> 未调用任何 AI 模型；项目虽集成 `@google/genai`，但本模块未实际用于打标。

#### 5.6.5 确认入库
- 「确认上传到置顶目录」：打标全部完成后才可点击。
- 生成的个人素材字段：名称（去扩展名）、分类（视频→`video`，其余→`gui`）、格式、大小、缩略图/预览、
  作者="当前用户"、来源="个人空间·置顶目录"、描述、标签、`uploadType`、`sourceFileName`、`uploadedAt`。
- 入库后：新素材前置插入列表，归属到置顶目录，自动选中并展开置顶目录，记录成功日志。
- 取消/关闭：作废当前上传会话（自增 session id，使正在进行的打标异步任务失效）。

## 6. 数据模型（关键字段）

## 7. 业务规则与约束清单（汇总）

1. 仅"项目空间 A"与"个人空间"展示素材库主体；项目空间 B / 共享空间为空态页。
2. 个人空间不提供素材详情弹窗与下载/导入；仅提供上传。
3. 上传仅支持 图片 / GIF / 视频；单次 ≤500 条且 ≤10GB；含不合规文件整批拦截。
4. 上传入库固定写入"个人空间·置顶目录（一级目录）"。
5. 删除文件夹连带子孙；其下素材上移到上级文件夹，若删除后无文件夹留存则解除归属。**文件夹目录允许删至 0 个**（空目录有新建引导与空态提示）。
6. 素材检索 = 当前文件夹范围（可选含子孙）× 分类 × 关键字（名称/标签），三者 AND。
7. 下载并发上限 3；超出排队；磁盘不足阻断。
8. 已下载素材再次点击 = 打开本地文件夹定位，而非重复下载。
9. DCC 导入需 DCC"已连接"；素材未下载则先自动下载再导入。
10. 导入兼容性由 格式 × DCC 矩阵决定；不兼容项禁用并显示原因。
11. 平面图片（PNG/JPG/JPEG）导入各 DCC 为直通；模型类导入 PS/Blender/Maya/3dsMax 走「可选」模式弹窗。
12. 切换空间/分类/关键字/文件夹/子文件夹开关 → 页码重置为 1。
13. 文件夹命名校验：非空、≤32 字符、禁用 `\ / : * ? " < > |` 与控制字符、不能以 `.` 开头、禁用 Windows 保留名（`CON/PRN/AUX/NUL/COM1–9/LPT1–9`）、同一父目录下不可同名；以上比较均大小写不敏感。

| 维度 | 要求 |
|---|---|
| 持久化 | 目录结构、归属、面板宽度写入 localStorage，刷新不丢失；读取容错、自愈 |

## 9. 边界与异常处理

| 场景 | 处理 |
|---|---|
| 选中的文件夹被删除 | 自动回退到主目录 → 根目录 → 首个文件夹 |
| 文件夹目录被删空（0 个） | 目录树显示"暂无文件夹，点击 + 新建"；素材区提示"请新建文件夹后查看"；受影响素材仅解除目录归属，数据不丢失 |
| 本地存储损坏/为空 | 回退到默认目录结构与默认归属；面板宽度回退 304px |
| 上传期间关闭面板 | 自增 session id 使异步打标任务失效，丢弃本次结果 |
| 同时重复触发同一素材下载 | 忽略后续触发 |
| 磁盘空间不足 | 阻断下载/导入下载，提示去设置页清理 |
| 不支持的文件类型/超量/超体积上传 | 整批拦截，分别给出针对性提示 |
| 搜索无结果 | 空态："没有找到匹配 "{关键字}" 描述的美术内容包" |
| 当前文件夹无子文件夹 | 提示"当前文件夹没有子文件夹" |

## 10. 待明确 / 可演进项（从代码推断的限制与 TODO）

1. **真实下载/磁盘**：当前为前端模拟进度与模拟磁盘容量；需接入真实分发与配额系统。
2. **真实 DCC 导入通道**：当前仅日志/弹窗模拟；需对接各 DCC 端口或脚本桥。
3. **真实 AI 打标**：当前为本地启发式；项目已集成 `@google/genai`，可演进为真实多模态打标。
4. **个人空间素材的下载/导入**：当前个人空间卡片不可点开详情、不能下载/导入；是否需要支持待定。
5. **项目空间 B / 共享空间内容**：V1 为空，仅空态；后续需支持预置/发布资产。
6. **视图切换**：代码中存在被注释的"卡片元信息显隐"开关（`showAssetCardInfo` 恒为 true），暗示曾计划提供视图模式切换。
7. **文件夹归属的可视化操作**：当前素材归属文件夹由默认规则/上传决定，UI 上未见"移动素材到其他文件夹"的交互（数据结构已支持）。
8. **多用户/权限**：当前为单用户本地演示（个人空间作者固定"当前用户"）；团队权限、审核、版本未涉及。
9. **PSD 等格式**：数据中存在 PSD/TGA 等格式素材，但兼容矩阵未覆盖（矩阵仅认 png/jpg/jpeg 为平面图），导入此类资产可能落入"不兼容"；需明确产品策略。
